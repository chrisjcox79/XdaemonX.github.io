Code injection on nonjailbroken iPhone

Using custom dylib to modify applications on nonjailbroken iPhone

Published Wed, Dec 9, 2020 by del0rean

Introduction

A few days ago I was writing about injecting FridaGadget.dylib inside application on nonjailbroken device so I was thinking why not to do the same thing, just with custom code (code injection).

The idea was to change some functionality of original application using dynamic library (dylib). Links for application are on my github and links are below.

  • https://github.com/XdaemonX/testApp
  • https://github.com/XdaemonX/testDylib

Like in my previous post, the same prerequsities apply.

Details about application

Three important files are:

  • TestClass.h
  • TestClass.m
  • ViewController.m

Implementation of ViewController inside ViewController.m file can be seen below.

@implementation ViewController

- (void)viewDidLoad {
    [super viewDidLoad];
    NSLog(@"ECHOView loaded");
    TestClass *test = [[TestClass alloc] init];
    [test whoami];
}

@end

Inside we have a method viewDidLoad which logs “ECHOView loaded” and after that we create an object of class TestClass which is defined in TestClass.h file. Then we call method whoami on it.

TestClass.h file:

@interface TestClass : NSObject
-(void)whoami;
@end

TestClass.m file:

@implementation TestClass
- (void)whoami {
    NSLog(@"ECHO I am from the app");
}
@end

After we run the app, in logs we can see something like this.

$ idevicesyslog -m ECHO
Dec  9 18:17:56 testApp[3774] <Notice>: ECHOView loaded
Dec  9 18:17:56 testApp[3774] <Notice>: ECHO I am from the app

Details about dynamic library

For our dynamic library, we have simple interface with one instance method in it. That is the method that will replace whoami method from interface TestClass.

testDylib.h file:

@interface testDylib : NSObject
- (void)replace;
@end

Implementation of this interface is where the real fun is. Let’s take a look at the code.

#import "testDylib.h"
#include <objc/runtime.h>

@implementation testDylib
static OriginalImpType originalImp;

+(void)load {
    // Just to log when the dylib gets loaded
    NSLog(@"ECHOLoading dylib");
    
    // We need to obtain Class objects of this class, and class of method which we will replace
    Class thisClass = [self class];
    Class toReplaceClass = NSClassFromString(@"TestClass"); // we could also use objc_getClass("TestClass")
    
    // Obtaining selectors for methods
    SEL selReplace = @selector(replace);
    SEL selToReplace = @selector(whoami);
    
    // We need to get instances of methods which will get substituted
    Method repMethod = class_getInstanceMethod(thisClass, selReplace);
    Method toReplace = class_getInstanceMethod(toReplaceClass, selToReplace);
    
    // impReplace will have implementation of -[testDylib replace] method 
    IMP impReplace = method_getImplementation(repMethod);
    
    // Set implementation of -[TestClass whoami] to be -[testDylib replace]
    method_setImplementation(toReplace, impReplace);
    
}

- (void)replace{
    NSLog(@"ECHO I am from dylib");
    
    originalImp(self, @selector(whoami));
}
@end
